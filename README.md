# 관심 후보 웹앱 — 휴대폰에서 보기

이평선 6단계 스캐너의 **관심 후보 추천**(괘상 건/리)을 휴대폰에서 즉시 보는 웹앱입니다.
서버 상주 없이 동작합니다 — GitHub Actions가 하루 4번 미리 스캔해 결과(`data/picks.json`)를
만들고, GitHub Pages가 그 정적 파일을 휴대폰에 보여줍니다. 비용 0원.

```
webapp/
├─ index.html              ← 휴대폰 웹앱 (이 파일이 화면)
├─ scanner_core.py         ← 분석 로직 (실행파일과 동일)
├─ scan_job.py             ← 병렬 스캔 → data/picks.json 생성
├─ requirements.txt
├─ data/picks.json         ← 스캔 결과 (Actions가 자동 갱신)
└─ .github/workflows/scan.yml  ← 하루 4회 자동 스캔
```

자동 스캔 시각(한국시간): **08:50 · 12:00 · 15:20 · 20:00** (월~금)

────────────────────────────────────────────
## 설치 (한 번만, 약 10분)
────────────────────────────────────────────

### 1. GitHub 저장소 만들기
1. https://github.com 로그인 → 우상단 **+** → **New repository**
2. 이름 예: `stock-picks` · **Public** 선택 · **Create repository**

### 2. 파일 올리기
이 webapp 폴더의 파일 전부를 저장소에 올립니다.
- 가장 쉬운 방법: 저장소 페이지에서 **Add file → Upload files** →
  `index.html`, `scanner_core.py`, `scan_job.py`, `requirements.txt`,
  `data` 폴더, `.github` 폴더를 드래그 → **Commit changes**
  (`.github` 폴더가 안 올라가면 따로 `.github/workflows/scan.yml` 경로로 추가)

### 3. GitHub Pages 켜기 (웹앱 주소 만들기)
1. 저장소 **Settings → Pages**
2. **Build and deployment → Source: Deploy from a branch**
3. **Branch: main / (root)** 선택 → **Save**
4. 1~2분 뒤 상단에 주소가 뜸:
   `https://<아이디>.github.io/stock-picks/`
   → 이 주소가 휴대폰 웹앱 주소입니다.

### 4. 자동 스캔 켜기
1. 저장소 **Actions** 탭 → 워크플로 사용 허용(Enable) 클릭
2. **scan** 워크플로 선택 → **Run workflow** 로 1회 수동 실행해
   `data/picks.json`을 채웁니다(첫 스캔, 2~4분).
3. 이후로는 매일 정해진 시각에 자동 실행됩니다.

### 5. 휴대폰 홈화면에 추가 (앱처럼)
- **아이폰(Safari)**: 주소 접속 → 공유 → **홈 화면에 추가**
- **안드로이드(Chrome)**: 주소 접속 → ⋮ → **홈 화면에 추가**

────────────────────────────────────────────
## 화면 설명
────────────────────────────────────────────
- 상단 탭: **단계별 건(乾)** / **4단계 리(離)**
- 종목 카드: 괘상 막대(5/20/50 표기) · 단계 · 종목명 · 시장 · 현재가 ·
  전일대비 · RSI · MFI · HA · MACD(★) · 시가총액 · 이전 단계 흐름
- 카드를 누르면 **차트**(캔들+이평선 5/20/50 · 거래량 · MACD)가 뜸 —
  Windows 실행파일과 동일 구성
- 우상단 **새로고침**으로 최신 결과 다시 불러오기

데이터는 FinanceDataReader 무료 데이터로 약 15~20분 지연입니다.

────────────────────────────────────────────
## 휴대폰에서 직접 스캔하기 (스캔 실행 버튼)
────────────────────────────────────────────
웹앱 우상단 **스캔 실행** 버튼을 누르면, 폰에서 바로 새 스캔을 돌릴 수 있습니다.
(실제 계산은 GitHub가 하고, 폰은 실행 요청만 보냅니다. 2~4분 뒤 자동 갱신.)

처음 한 번만 GitHub 토큰 입력이 필요합니다.
1) GitHub → 우상단 프로필 → Settings → Developer settings
   → Personal access tokens → **Tokens (classic)** → Generate new token (classic)
2) Note: 아무거나 / Expiration: 길게(90일~No expiration) /
   Scopes: **repo** 체크 → Generate token → ghp_... 복사
3) 웹앱에서 **스캔 실행**을 누르면 토큰 입력창이 뜸 → 붙여넣기
   → 이 기기에 저장되어 다음부터는 안 물어봄.

버튼 동작:
- 요청 성공 → "스캔 시작됨 · 결과 기다리는 중…" → 끝나면 자동으로 최신 반영
- 401 → 토큰 인증 실패(토큰 다시 입력) / 403·404 → repo 권한·저장소 이름 확인

⚠️ 주의: 이 웹앱은 정적 페이지라 토큰을 브라우저(이 기기)에만 저장합니다.
   본인만 쓰는 용도로 두고, 웹앱 주소를 토큰과 함께 남에게 공유하지 마세요.

### 새 스캔 전까지 기존 자료 보존
데이터 수신이 실패하거나 분석 종목이 너무 적으면, 기존 결과를 덮어쓰지 않고
그대로 유지합니다. 즉 새 스캔이 정상 완료되기 전까지 직전 결과가 계속 보입니다.

────────────────────────────────────────────
## 스캔 속도·범위 조정
────────────────────────────────────────────
`.github/workflows/scan.yml`의 env 값으로 조절합니다.
- `TOP_KOSPI` / `TOP_KOSDAQ` : 시총 상위 몇 종목까지 볼지 (기본 380/420 ≈ 800종목)
- `MAX_WORKERS` : 동시 다운로드 수 (기본 24, 늘리면 빨라지나 데이터 서버 부하)

종목 수를 줄이면 스캔이 더 빨라집니다. 800종목·워커24 기준 보통 2~4분입니다.

스캔 시각을 바꾸려면 `scan.yml`의 cron(UTC 기준)을 수정하세요.
한국시간 = UTC + 9 (예: 09:00 KST = 00:00 UTC).

## 로컬에서 테스트 (선택)
────────────────────────────────────────────
```
pip install -r requirements.txt

python -m http.server 8000    # 브라우저에서 http://localhost:8000 접속
```
